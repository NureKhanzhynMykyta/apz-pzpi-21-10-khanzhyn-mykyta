1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)
1.1 Background (Передумови)

В сучасних умовах безпека та контроль доступу в навчальних закладах стають все більш актуальними завданнями. Зростаючі вимоги до безпеки, зокрема підвищений рівень свідомості щодо безпеки у навчальних середовищах та реальні загрози, пов'язані зі зловживанням доступом, створюють необхідність у вдосконаленні систем контролю доступу та ведення обліку присутності. Недостатня ефективність існуючих методів контролю та обліку приводить до пошуку нових, інноваційних рішень, які забезпечать надійний та ефективний контроль доступу та точне відображення присутності учасників навчального процесу.

1.2 Business Opportunities (Бізнес – можливості)

На ринку ідентифікаційних систем для навчальних закладів існують кілька конкуруючих програм, проте багато з них мають обмежені функціональні можливості або не відповідають сучасним вимогам безпеки та зручності використання.
Програми для ідентифікації особистості - Деякі компанії пропонують спеціалізовані програмні рішення для ідентифікації особистості в навчальних закладах. Наприклад, компанія "SecureEd" надає систему, яка поєднує в собі NFC-технології та мобільний застосунок для ідентифікації осіб та ведення обліку присутності. Однак, їхній продукт не має інтеграції з існуючими системами управління навчальним процесом та може бути менш зручним у використанні порівняно з запропонованою програмою.
Системи керування навчальним процесом - Деякі LMS, наприклад, Moodle або Blackboard, включають в себе деякі функції контролю доступу та обліку присутності. Наприклад, LMS університету "XYZ" дозволяє викладачам створювати списки присутності для кожного заняття. Проте ці системи часто мають обмежені можливості в ідентифікації осіб та надійності обліку присутності.
Таким чином, Програмна система для ідентифікації особистості в навчальних закладах  відповідає ринковим потребам, надаючи точний та зручний засіб контролю доступу та обліку присутності, і має конкурентні переваги порівняно з існуючими програмами.


2 VISION OF SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1 Vision Statement (Окреслення концепції)

Програмна система для ідентифікації особистості в навчальних закладах пропонує інноваційний підхід до ідентифікації особистості в навчальних закладах, спрямований на створення безпечного та ефективного середовища для учнів і персоналу. Ми бачимо світ, де навчальні заклади можуть безперешкодно впроваджувати технології для забезпечення безпеки та відслідковування присутності без надмірних труднощів та витрат. Ця система спростить процеси контролю доступу та забезпечить точну та швидку ідентифікацію особистості, що забезпечить необхідний рівень безпеки та ефективності для навчальних закладів.
Цей продукт відповідає потребам різних клієнтів, включаючи учнів, викладачів та адміністраторів навчальних закладів. Він дозволяє забезпечити безпеку та контроль доступу до навчальних приміщень, забезпечуючи водночас зручність та ефективність використання для всіх сторін.
Ця програмна система не лише спростить процеси ідентифікації, але й покращить загальний досвід користувачів та забезпечить надійність та безпеку даних. Вона відповідає стратегічним напрямкам розвитку навчальних закладів, пропонуючи інтегроване та ефективне рішення для сучасних вимог у сфері безпеки та ідентифікації особистості.

2.2 Major Features (Головна функціональність)

MF-1: Реєстрація учнів/студентів
• Можливість організувати класи та зареєструвати учнів/студентів через систему управління.
MF-2: Контроль доступу:
• Користувачі можуть ідентифікувати себе за допомогою особистих NFC брелків або за допомогою NFC на телефоні через мобільний застосунок.
• Система перевіряє ідентифікаційні дані користувача при вході в навчальний заклад та відкриває доступ до приміщення, запобігаючи несанкціонованому доступу.
MF-3: Відстеження присутності:
• Фіксація часу приходу та виходу кожного користувача, зберігання інформації про присутність в базі даних, доступній для перегляду у особистому кабінеті.
MF-4: Особистий кабінет:
• Кожен користувач має особистий кабінет для перегляду історії відвідувань, розкладу занять, оцінок та інших особистих даних.
MF-5: Інтеграція з NFC-брелками:
• Розробка NFC-брелків з унікальними ідентифікаторами для кожного користувача, що дозволяє швидку та зручну ідентифікацію.
MF-6: Мобільний застосунок для користувачів:
• Створення мобільного додатка для зручного доступу до особистого кабінету, відстеження присутності та отримання сповіщень.
MF-7: Адміністративний інтерфейс:
• Інтерфейс для адміністраторів навчального закладу для керування даними користувачів, налаштування системи та перегляду звітів.
MF-8: Система сповіщень:
• Можливість надсилання мобільних повідомлень і сповіщень користувачам щодо важливих подій та інформації.
MF-9: Аналітика та звітність:
• Збір та аналіз даних про присутність для створення звітів та аналізу ефективності системи.
MF-10: Інтеграція з іншими системами:
• Можливість інтеграції з іншими системами навчальних закладів, такими як система електронного журналу та система безпеки.

3. SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ)
3.1 Scope of Initial Release (Рамки первинного випуску)

Програмна система для ідентифікації особистості в навчальних закладах буде наділена базовим функціоналом для ефективного контролю доступу та відстеження присутності користувачів. Основні функції та характеристики якості, заплановані для початкового випуску продукту, виглядають наступним чином:
Серверна частина:
• Авторизація та реєстрація користувачів з використанням безпечних методів аутентифікації.
• Керування інформацією про користувачів, включаючи студентів/учнів, викладачів та інший персонал навчального закладу.
• Зберігання даних про присутність користувачів та їхню активність в базі даних.
Клієнтська частина:
• Авторизація та реєстрація користувачів, включаючи студентів та викладачів.
• Відображення особистого кабінету для кожного користувача з можливістю перегляду історії відвідувань та розкладу занять.
• Отримання повідомлень про статус присутності та подій в навчальному закладі.
Мобільний застосунок:
• Авторизація та реєстрація користувачів в мобільному додатку.
• Перегляд особистого кабінету для кожного користувача з можливістю відстеження присутності та отримання повідомлень про статус.
• Оповіщення про події в навчальному закладі та зміни в розкладі занять.
IoT/Smart Device:
• Збір та передача даних про присутність та активність користувачів у приміщеннях.
• Інтеграція пристроїв IoT з серверною частиною системи для забезпечення безпеки даних та надійності збереження інформації.

3.2 Scope of Subsequent Releases (Рамки наступних випусків)

У подальших випусках програмної системи для ідентифікації особистості в навчальних закладах планується поетапне розширення функціональності для забезпечення більшої зручності та безпеки користувачів, підвищення ефективності та привабливості продукту. Основні функції, які будуть відкладені до наступних випусків, включають:
Серверна частина:
• Розширення можливостей аналітики для забезпечення більш детальної статистики про присутність та активність користувачів.
• Інтеграція з додатковими сервісами для розширення функціональності та покращення якості обслуговування.
Клієнтська частина:
• Розширення інтерактивної картографічної системи для надання додаткової інформації про навчальний заклад, його історію та культурні особливості.
• Впровадження можливості відстеження та оцінки якості обслуговування та забезпечення зворотного зв'язку від користувачів.
Мобільний застосунок для студентів/учнів та співробітників:
• Розширення можливостей оплати, включаючи можливість оплати послуг та товарів прямо через застосунок.
• Підтримка мовного перекладу для іноземних користувачів для полегшення їхнього взаємодії з системою.
Мобільний застосунок для адміністраторів:
• Розширення функціоналу для забезпечення кращого планування та управління ресурсами навчального закладу.
• Можливість аналізу реакції користувачів на події та інформацію для вдосконалення сервісу.
IoT/Smart Device:
• Розширення інтеграції з розумними транспортними засобами для надання додаткових послуг та підвищення рівня комфорту під час переїзду.
• Забезпечення безпеки та конфіденційності даних, зокрема в умовах втрати чи крадіжки пристроїв ідентифікації.

4 BUSINESS CONTEXT (БІЗНЕС – КОНТЕКСТ)
4.3 Operating Environment (Робоче середовище)

Технології та програмні засоби, які використовуються в проєкті " Програмна система для ідентифікації особистості в навчальних закладах ":
1. Веб-додаток:
• Мова програмування: Веб-додаток буде розроблено з використанням мов програмування C# для серверної логіки. Веб-додаток використовуватиме HTML, CSS і JavaScript для клієнтського інтерфейсу.
• Фреймворк: Для створення серверної частини веб-додатка буде використовуватися фреймворк ASP.NET.
• База даних: Використовуйте систему управління базами даних (СУБД) для зберігання і обробки даних. Наприклад, буде використано Microsoft SQL Server або іншу СУБД.
2. Мобільні застосунки (Android та iOS):
• Мови програмування: Для розробки мобільного застосунка для платформи Android, можна використовувати мови програмування Java або Kotlin. Для розробки застосунків для платформи iOS, використовуватиметься Swift або Objective-C.
• Бібліотеки: В мобільних застосунках для взаємодії з сервером Microsoft SQL Server використовуватиметься відповідні бібліотеки. Наприклад, для платформи Android можна використовувати бібліотеку JDBC або Microsoft JDBC Driver для SQL Server. Для платформи iOS можна використовувати бібліотеки, такі як Microsoft SQL Server для Swift або Objective-C, або інші бібліотеки доступу до бази даних, які підтримують з'єднання з Microsoft SQL Server.
• Фреймворки: Для розробки під Android можна використовувати Android Studio разом з Android SDK. Для розробки під iOS використовуватиметься Xcode.
• Операційні системи: Застосунки будуть спрямовані на операційні системи Android і iOS.
3. Інші технології:
• IoT та NFC: Для зчитування даних з NFC-брелків та інтеграції з IoT-пристроями (розумні транспортні засоби), використовуватимуться відповідні технології та програмні бібліотеки.
4. Вимоги:
• Доступність: Веб-додаток повинен бути доступний з різних пристроїв та браузерів. Система повинна бути доступною для користувачів у визначених географічних областях, де планується надання послуг.
• Надійність: Всі компоненти системи повинні бути стійкими до відмов та забезпечувати надійну роботу в умовах навантаження.
• Продуктивність: Система повинна працювати з високою продуктивністю, забезпечуючи швидкий доступ користувачів до інформації та послуг.
• Цілісність даних: Забезпечити цілісність даних, особливо при обміні даними між компонентами системи.
